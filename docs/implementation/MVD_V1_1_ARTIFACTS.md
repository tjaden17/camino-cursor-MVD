# MVD v1.1 — artefact paths and owners

**Purpose:** Map unified AC items to **where files live** and **who owns** them (pipeline, QC UI, prompts, docs). Aligns with Decisions 9–10 and 12 in `plans/implementation/IMPLEMENTATION_PLAN_MVD_V1_1.md`.

## Original vs processed


| Kind                                  | Location                                                                                      | Owner                                      |
| ------------------------------------- | --------------------------------------------------------------------------------------------- | ------------------------------------------ |
| Original onboarding CSV (PO-supplied) | `data/onboarding/*.csv`                                                                       | Human / import                             |
| Processed onboarding JSON             | `data/onboarding/*-onboarding-derived.json`, `data/user-onboarding/*-onboarding-derived.json` | `npm run onboarding:csv` or manual         |
| KPI spec (product-defined contract)   | `data/kpi-spec/kpi-spec-v1.json`                                                              | Product + CI (`npm run validate:kpi-spec`) |
| Zoho / custom extracts                | `data/zoho/`, `data/custom/`                                                                  | Human drop + future normalisers            |


## Pipeline outputs (`out/`, gitignored)


| Artefact                      | Description                                                  |
| ----------------------------- | ------------------------------------------------------------ |
| `out/pipeline-run.json`       | Stage manifest                                               |
| `out/org-context.json`        | Merged org snapshot (role-based tie-break)                   |
| `out/strategy-catalogue.json` | **Canonical** 12 KPIs + 6 decisions + 3+3 sample calc labels |
| `out/agent-signals.json`      | Agent KPI selections                                         |
| `out/processed-signals.json`  | Preview deck signal cards                                    |
| `out/review-gates.json`       | Card-quality gates                                           |
| `out/review-gates-v11.json`   | v1.1 structure gates (`npm run review:v11`)                  |


## Mirrors (optional commit)

Generated by `npm run pipeline -- --skip-llm` (or full LLM run):


| Path                                            | Role                                                     |
| ----------------------------------------------- | -------------------------------------------------------- |
| `data/org/{orgId}.json`                         | Org merge + embedded `strategyCatalogue` for file review |
| `data/onboarding/{userId}-strategy-mirror.json` | Per-user slice of strategy catalogue                     |


These paths are **gitignored by default**; remove the ignore entries if you want them in version control.

## Demo path (hybrid UX)

1. Run `npm run pipeline -- --skip-llm`.
2. Open signal **preview** via existing preview flows (representative card subset).
3. Open **Org context** for the full **12 KPIs + 6 decisions** and merged org JSON.
