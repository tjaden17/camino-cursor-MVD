Instructions to build Minimum Viable Data


Goal
I want to test if AI can product the data I want. My main goal is the do the ‘Quality Check the dat provided by AI’


Let’s call this release the ‘MVD -Minimum viable data’. Ultimately, I want to create a product (a data/decision intelligence tool) that helps surface insights from a company’s data. But for now, we don’t need to consider the user experience. I just want to see the quality of the data AI can produce.


Description
Test if AI can produce the data I want. The data the AI will ingest includes 1) user/company information (collected from onboarding) 2) the data from the company (provided by the users by csv, in future it’ll be API/integrations) 3) publicly available information (e.g. publicly available benchmark data).


The processed data is to be saved to a database. And rendered visually for the user.


The key actions for the user are scroll (scroll through different KPI overviews) and explore (user can explore in depth analysis and synthesis for a kpi) and receive weekly brief (get a weekly update on what’s important)


Acceptance Criteria (user perspective) (for 25 Mar build)
* Surge & Sam can provide onboarding information to me (via a call). Which is saved, normalised, stored by the system (e.g. user onboarding files).
* Surge & Sam can view 3 requested signals via the signal preview web ui. No login required. On  my computer.. With 3 key interactions: 1-Signal card overview. 2-Signal card expanded. 3-View different signal card.
* On the signal preview screen, I can toggle between Surge & Sam view
* Surge & Sam can view 3 recommended signals (with sufficient data). With 3 key interactions.1-Signal card overview. 2-Signal card expanded (including why it’s recommended). 3-View different signal card.
* Surge & Sam can view 3 recommended signals (with insufficient data). 1-Signal card overview. 2-Signal card expanded (why it’s recommended, what data is missing, tips for practically sourcing the date). 3-View different signal card.
* The data flow & architecture, from ingesting onboarding files and user data files, to agent processing, agent outputs, is explained clearly. Is designed in a modular way where it it easy to amend instructions in order to fine tune the data Surge & Sam sees

	Name: Weekly Brief


User goal: Given 1min at beginning of the week, what does a CEO need to care about?
	Section 1: Signals this week


**Each row contains:**
- KPI name
- Current value (absolute)
- Delta from last period (arrow + number)
- Status badge: `improving` / `stable` / `watch` / `alert`


**Status badge rules:**
| Badge | Condition |
|---|---|
| `improving` | Delta > threshold in positive direction |
| `stable` | Delta within ±threshold |
| `watch` | Delta > threshold in negative direction, not yet critical |
| `alert` | Delta > 2x threshold in negative direction, or threshold crossed |


Section 2: Top 3 issues this week


The top 3 most important signal this week gets one paragraph of interpretation each.
**Rules:**
- Maximum 5 sentences
- Must reference the trend (not just this week's number)
- Must give one forward-looking statement ("watch whether X")
- Must not recommend an action — that's the exec's job

Strategy Consultant
Expertise: Metric prioritization (for a particular function/role), business strategy, decision frameworks
How they help:
* Recommends the metrics a user should check weekly (e.g. CEO, CSM)
* Connects each metric to a specific business decision
* Considers industry, stage, team size, and priorities
* Filters out vanity metrics in favor of actionable ones
* Thinks in terms of what drives business outcomes


Data Engineer agent
Expertise: Data pipelines, ETL, schema mapping, data contracts, integration
How they help:
* Checks column mappings are correct
* Validates data types and parsing logic
* Identifies edge cases in source data
* Designs robust data contracts
* Ensures schema validation
* Thinks in terms of data flow and transformation


Data analyst agent
Expertise: Data modeling, SQL optimization, analytics, metrics, reporting
How they help
* Optimizes database structure
* Writes efficient SQL queries
* Identifies data quality issues
* Suggests meaningful metrics
* Designs aggregation strategies




Business intelligence agent
Expertise: Business intelligence, metric calculations, SQL, data validation
How they help:
* Validates formulas and calculations
* Thinks through SQL logic and edge cases
* Identifies missing filters or wrong assumptions
* Checks date range handling
* Ensures nulls and duplicates are handled correctly
* Always asks to see raw data before confirming numbers


Management Consultant Agent
Expertise: SMB operations, metric prioritization, business strategy, decision frameworks

* Recommends the 5 metrics a CEO should check weekly

Draft Critical Path / Overall workflow to create data
* User 1 (exec user) provides their information via onboarding. Saved to ‘Exec user file’.
* User 2 (manager user) provides their information via onboarding. Saved to ‘CSM user file’
* User 2/Admin connects their data. Data files saved to system. 
* AI agents work, in parallel, or in succession, in to save data to database, enough to produce data required in main use cases and key user views.
* At certain steps, e.g. after data is ingested for the first time for a data shape, there ia a Human in the loop. E.g. Where a user verifies normalised columns names. So that user can trust the data hygiene, normalisation.
* Data is rendered visually to create signal cards, so that user can scroll (in scroll view) and deep dive (when they explore a signal card)
* Human in the loop - User can provide feedback on usefulness of the processed data, i.e. data in the signal cards, manually via a call.
